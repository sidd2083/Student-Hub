# Student Hub

A Firebase-backed React+Vite study platform for Grade 9–12 students in Nepal. Features study notes, past exam papers (PYQ), Nep AI assistant, Pomodoro timer with Firestore study-time tracking, streak system, Report Card analytics, and a Firestore-based leaderboard.

## Run & Operate

```bash
# Start frontend (workflow: "Start application")
PORT=22581 BASE_PATH=/ pnpm --filter @workspace/student-hub run dev

# Start backend (workflow: "Backend API") — must run alongside frontend
PORT=8080 DATABASE_URL=$DATABASE_URL node --enable-source-maps artifacts/api-server/dist/index.mjs

# Build backend (required after any api-server source changes)
pnpm --filter @workspace/api-server run build

# Push DB schema (run once or after schema changes)
pnpm --filter @workspace/db run push-force

# Build frontend for production
pnpm --filter @workspace/student-hub run build
```

Required env vars:
- `VITE_FIREBASE_API_KEY`, `VITE_FIREBASE_AUTH_DOMAIN`, `VITE_FIREBASE_PROJECT_ID`, `VITE_FIREBASE_STORAGE_BUCKET`, `VITE_FIREBASE_MESSAGING_SENDER_ID`, `VITE_FIREBASE_APP_ID`, `VITE_FIREBASE_MEASUREMENT_ID`
- `DATABASE_URL` (auto-set by Replit PostgreSQL)

## Stack

- **Frontend**: React 18 + Vite 7, TypeScript, Tailwind CSS v4, shadcn/ui
- **Routing**: wouter
- **State**: TanStack Query v5
- **Backend API**: Express 5 + Drizzle ORM + PostgreSQL (`artifacts/api-server/`) — handles Notes, PYQs, Tasks
- **Auth & Realtime DB**: Firebase Auth (Google sign-in) + Firestore
- **Storage**: Firebase Storage (for note/PYQ file uploads)

## Where things live

- `artifacts/student-hub/src/pages/` — all page components
- `artifacts/student-hub/src/components/` — Layout, ErrorBoundary, SoftGate, ProtectedRoute, AppShell
- `artifacts/student-hub/src/context/AuthContext.tsx` — Firebase auth + Firestore profile + redirect logic
- `artifacts/student-hub/src/lib/firebase.ts` — Firebase init (reads VITE_FIREBASE_* env vars)
- `artifacts/api-server/src/routes/` — Express route handlers (notes, pyqs, tasks, users, ai, scores, health)
- `lib/api-client-react/src/generated/api.ts` — TanStack Query hooks (auto-generated by orval from OpenAPI spec)
- `lib/db/src/schema/` — Drizzle schema (notes, pyqs, tasks, users, mcqs, scores)
- `lib/api-spec/openapi.yaml` — OpenAPI spec (source of truth for API contracts)
- `artifacts/student-hub/vite.config.ts` — Vite config; proxies `/api` → `http://localhost:8080`

## Architecture decisions

- **Two workflows required**: Vite frontend (port 22581) + Express backend (port 8080). Vite proxies `/api/*` to port 8080 in dev.
- **MCQ system fully removed** from frontend — `McqPractice.tsx` file exists but has no active route.
- **Firestore for user data** — streak, studyTime, lastActive, grade, role in `users/{uid}`. Logs in `users/{uid}/studyLogs/{date}`.
- **PostgreSQL for content** — Notes, PYQs, Tasks stored in PostgreSQL via Drizzle ORM.
- **Array.isArray() guards** on every API hook result before `.map()/.filter()` — prevents crashes when API returns non-array.
- **ErrorBoundary** wraps the entire app (in App.tsx) and inner Router to catch runtime crashes gracefully.
- **SoftGate pattern** — unauthenticated users see a blur overlay prompting Google sign-in on tool pages.

## Product

- Public: Home, Notes list, Note viewer, PYQ list, PYQ viewer, About, Contact
- Auth-required: Dashboard, Report Card (/report), Pomodoro, To-Do, Nep AI, Leaderboard, Settings
- Admin (/admin): Manage Notes, PYQs, Announcements, Users, Study Reports, SEO Panel

## User preferences

- Nepal students Grade 9–12 audience
- Firebase-first for user data (not PostgreSQL)
- No MCQ system — removed completely
- Report Card replaces MCQ in navigation and admin

## Gotchas

- **Both workflows must be running** — frontend crashes silently without the backend API server
- Vite config throws if `PORT` or `BASE_PATH` env vars missing — always include them in workflow command
- After changing `artifacts/api-server/src/`, must `pnpm --filter @workspace/api-server run build` then restart "Backend API" workflow
- After changing `lib/db/src/schema/`, must `pnpm --filter @workspace/db run push-force` to apply to DB
- Always use `Array.isArray(data) ? data : []` before `.map()/.filter()` on TanStack Query results
